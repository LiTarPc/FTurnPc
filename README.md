## Как это работает

Приложение запускает фоновый процесс `freeturn-client`, который поднимает туннель на локальном адресе `127.0.0.1:9000`. Приложение создаёт системный виртуальный сетевой интерфейс WireGuard (wg-turn) и перенаправляет весь системный интернет-трафик в этот туннель. 

Все пакеты упаковываются в RTP-медиапотоки с шифрованием и передаются на TURN-серверы VK. С точки зрения провайдера ваше соединение выглядит как обычный групповой звонок VK.

```
Системный трафик → WireGuard (wg-turn) → FTurn Client (127.0.0.1:9000) → RTP/DTLS (Внешний интернет) → VK TURN → Ваш VPS (freeturn-server)
```

Приложение имеет защиту от зацикливания маршрутов: IP-адреса серверов VK и TURN динамически определяются из логов соединения на старте и добавляются в системную таблицу исключений Windows, чтобы служебный трафик обходил туннель напрямую.

## Быстрый старт (Windows)

1. Скачайте последнюю версию **`FTurnPc.exe`** из релизов.
2. Запустите приложение **от имени администратора** (это необходимо для создания виртуального сетевого интерфейса WireGuard и настройки маршрутов).
3. Добавьте сервер кнопкой `+` вставив ссылку формата `freeturn://...`.
4. Для изменения количества потоков или добавления ссылки на звонок нажмите на шестерёнку настроек профиля.
5. Нажмите кнопку включения по центру для активации туннеля.

## Формат ссылки

Приложение принимает конфигурационные ссылки в формате Base64-строки с префиксом:

```
freeturn://<Base64-encoded-JSON>
```

Декодированный JSON содержит конфигурацию подключения:
```json
{
  "name": "Название сервера",
  "provider": "vk",
  "peer": "IP:порт_TURN_сервера",
  "transport": "udp",
  "obf": "rtpopus...",
  "key": "ключ_обфускации",
  "cid": "client_id",
  "wg": "[Interface]\nPrivateKey = ...\nAddress = 10.0.0.2/24\n...\n[Peer]\nPublicKey = ...\nAllowedIPs = 0.0.0.0/0",
  "links": "https://vk.ru/call/join/..."
}
```

Ссылку можно добавить как через меню `+`, так и просто нажав **Ctrl+V** в любом месте окна приложения.

## Сборка из исходников

Для корректной работы с 32-битным драйвером `wintun.dll` приложение собирается под архитектуру `windows/386` (32-бит).

**Требования:** Go 1.26+, Node.js 22+, Wails v2.

```bash
# Клонирование репозитория
git clone https://github.com/luminescq/PWDTT.git
cd PWDTT

# Сборка проекта (флаг -skipbindings пропускает генерацию типов, если вы не меняли Go-интерфейсы)
wails build -platform windows/386 -skipbindings
# → Готовый файл соберется в build/bin/FTurnPc.exe
```
